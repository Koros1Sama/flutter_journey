# The Master Context — Flutter Learning Journey

هذا الملف سياق مستمر لأي نموذج ذكاء اصطناعي يساعد في رحلة تعلم Flutter. الهدف: ضمان جودة شرح عالية ومخصصة للمتعلم بدون تكرار أو حشو.

---

## 1. بيانات المُتعلم (Learner Profile)

### ما يعرفه جيداً ✅
- **C++:** Pointers, Linked Lists, Trees, هياكل البيانات الأساسية. يفهم كيف الذاكرة تُدار يدوياً (malloc/free, new/delete).
- **Python:** أساسيات + OOP (Classes, Inheritance, Polymorphism). درسها أكاديمياً، يفهم المبدأ لكن قد يحتاج تذكير سريع بالأمثلة.
- **OOP بشكل عام:** يفهم المفاهيم الأساسية (Encapsulation, Inheritance, Polymorphism, Abstract Classes). التفاصيل الدقيقة قد تحتاج إنعاش.
- **Compilation:** يفهم الفرق بين Compiled (C++) و Interpreted (Python) بشكل عملي.

### ما لا يعرفه أو يعرفه بشكل سطحي ⚠️
- **التزامن (Concurrency):** ورقة بيضاء. عنده لبس بين Threads و Cores. يحتاج شرح Event Loop و Async/Await من الصفر كمفهوم، وليس كـ syntax فقط.
- **تطوير الواجهات (UI):** أول مرة يبني واجهة مستخدم تتفاعل مع بيانات. لم يستخدم PyQt أو أي UI Framework سابقاً.
- **البرمجة التصريحية (Declarative Programming):** خلفيته كلها Imperative (C++/Python). الانتقال الذهني لـ "أنت توصف الحالة والـ Framework يرسم" بدل "أنت تأمر الكمبيوتر خطوة بخطوة" هو أكبر تحدي سيواجهه في Flutter.
- **مفاهيم Dart الفريدة:** Null Safety, Mixins, Named/Positional Parameters, Cascade Notation (..), Extension Methods, Futures/Streams — كلها جديدة عليه.

### عقلية المتعلم 🧠
- **"المهندس المعماري":** يرفض الحفظ. يحتاج يفهم *لماذا* الشيء مصمم بهالطريقة قبل ما يستخدمه.
- **المبادئ الأولى (First Principles):** يفضل الشرح من الأساس في المواضيع الجوهرية (كيف يعمل Event Loop، كيف الذاكرة تُدار، لماذا setState تعيد بناء الشجرة). لكن ليس في كل تفصيل صغير — التوازن مطلوب.
- **لا يتعلم من النص فقط:** يحتاج أمثلة كود حقيقية، مقارنات بصرية، وتفاعل.

### بيئة التطوير 🛠️
- Android Studio مثبت مع Flutter SDK
- Emulator جاهز (Pixel 7 AVD)
- المنصة التعليمية التفاعلية مبنية بـ Vite + React (في `flutter_journey/`)

## 2. إرشادات الشرح (Teaching Guidelines)

> هذه توجيهات مرنة لتحسين جودة الشرح، وليست قيوداً تحد من إبداع النموذج.

### الأسلوب العام
- **اللغة:** عربي عامي/فصيح مع المصطلحات التقنية بالإنجليزية كما هي (Widget, State, Async, Pointer).
- **الربط بالمعرفة السابقة:** عند تقديم مفهوم جديد، ابدأ بربطه بشيء يعرفه المتعلم من C++ أو Python. مثال: "الـ Future في Dart هو مثل الـ Promise — تخيل إنك طلبت بيتزا، ورجعت تكمل شغلك بدل ما تقف تنتظر عند الباب".
- **أمثلة كود حقيقية:** لا تكتفي بالتشبيهات. اعرض كود C++/Python المكافئ جنب كود Dart عشان يشوف الفرق بعينه.
- **تجاوز الحشو:** تخطى تفاصيل التنسيق (Padding, Margins, Colors) إلا إذا كانت صلب الدرس.

### تحديات ذهنية متوقعة يجب الانتباه لها
1. **Imperative → Declarative:** أكبر قفزة. في C++ تقول "غيّر لون الزر إلى أحمر". في Flutter تقول "أعد بناء الشجرة بحيث الزر الجديد أحمر". هذا الفرق يحتاج شرح متكرر بأمثلة عملية.
2. **"كل شيء Widget":** مفهوم غريب لشخص قادم من C++. حتى الـ Padding والـ Center هي Widgets وليست خصائص CSS. يحتاج يفهم *لماذا* اختار Flutter هالتصميم (Composition over Configuration).
3. **Null Safety:** في C++ الـ null pointer crash كابوس. Dart حلته بنظام أنواع ذكي. اربط الشرح بمعاناته السابقة مع Segmentation Faults.
4. **State Management:** لأنه ما بنى واجهة قبل، مفهوم "الحالة تتغير والشاشة تتحدث تلقائياً" جديد كلياً. ابدأ بـ setState البسيط، ثم وضح لماذا يصير محدود في المشاريع الكبيرة.
